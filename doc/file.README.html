Decidim KS

Free Open-Source participatory democracy, citizen participation and open government for cities and organizations

This is the open-source repository for decidim_warszawa_ks, based on Decidim.

Setting up the application

You will need to do some steps before having the app working properly once you've deployed it:

  1. Open a Rails console in the server: bundle exec rails console
  2. Create a System Admin user: ruby user = Decidim::System::Admin.new(email: <email>, password: <password>, password_confirmation: <password>) user.save!
  3. Visit <your app url>/system and login with your system admin credentials
  4. Create a new organization. Check the locales you want to use for that organization, and select a default locale.
  5. Set the correct default host for the organization, otherwise the app will not work properly. Note that you need to include any subdomain you might be using.
  6. Fill the rest of the form and submit it.

You're good to go!

Dokumentacja kodu źródłowego

Kod źródłowy oprogramowania został udokumentowany w formie komentarzy do klas, funkcji i innych składowych systemu. Bazowym językiem stosowanym w komentarzach jest język angielski, ponieważ framework Ruby On Rails oraz rozwiązanie Decidim jest napisane w języku angielskim a ponadto wynika to też z dobrych praktyk programistycznych.

Dokumentacją został objęty kod wytworzony na potrzebę tego projektu w szczególności te obszary, w których zawarta jest logika biznesowa systemu np. gem projects, gem processes.

Środowiskiem programistycznym wykorzystanym przy implementacji systemu jest Ruby On Rails. Istotną wartością tego frameworku jest samodokumentujący się kod źródłowy wynikający z tego w jaki sposób są nazywane funkcje, jak intuicyjna i przejrzysta jest składnia oraz jak naturalnie pisany jest kod by był zrozumiały dla ludzi. W związku z tym została przyjęta zasada, że dokumentacji nie podlegają oczywiste i zrozumiałe linie kodu (np. metoda ujęta w jednej linii). Również z tego powodu by nie robić nadmiarowych tekstów utrudniając programiście pracę nad rozwojem oprogramowania.

Biblioteki zależnie takie jak np. Decidim posiadają własną dokumentację. Dokumentacja tych bibliotek nie wchodzi w zakres dokumentacji projektowej. Aby zapoznać się z dokumentacją oprogramowania stron trzecich należy odwiedzić ich oficjalne strony internetowe.

Wygenerowana dokumentacja znajduje się w katalogu /doc projektu. Aby móc się z nią zapoznać należy uruchomić plik doc/index.html w dowolnej przeglądarce internetowej. Używanie dokumentacji nie wymaga dostępu do Internetu.
